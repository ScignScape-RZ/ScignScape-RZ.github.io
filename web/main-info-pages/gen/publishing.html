Publishing

Digital text and resources have profoundly altered the nature of academic and scientific publishing — indeed, the full effects of these changes are still only beginning to be felt. Authors and publishers have not yet discovered how to truly leverage the new opportunities afforded by embedding publications in a digital/computational ecosystem. These possibilities include a more interactive/multimedia reading experience (with respect to conventional published text) as well as a more transparent and rigorous data-sharing ecosystem. LTS contributes certain new code and protocols in both of these areas.

In terms of digital text, LTS's Hypergraph Text Encoding Protocol (HTXN) (also discussed in the software language engineering page) can serve as the basis for a more advanced interactive reading experience, via more advanced e-reader software for formats such as e-books and PDF. LTS's custom markup language can generate PDF documents (via LaTeX) embedded with extra information about textual/discursive features such as sentence boundaries, named entities/keywords and phrases, or any content which requires special display (figure illustrations, graphics, charts, etc.). A combination of preprocessors during LaTeX generation and LaTeX auxiliary files allows information about textual artifacts to be notated both in a linguistic context (modeling discourse in terms of words, sentences, paragraphs, etc.) and with respect to screen and PDF page coordinates (e.g., allowing readers to select a sentence by clicking on or near a word within the sentence).

In the context of data sharing, LTS tools can help authors (or software developers on authors' behalf) package scientific data with custom applications to promote interactive, scientifically rigorous examination (and potential reuse/reanalysis) of authors' data (and research methods or laboratory protocols). Many published data sets currently use generic formats such as spreadsheets or .npy (numeric python) files to encode and distribute their open-access data. Such generic formats have limited GUI capabilities and also are less usable for subsequent analysis, because those utilizing the shared data may need to write additional computer code to import the data into an analytic environment. By contrast, publishing data sets according to more recents standards (e.g., Research Objects) often entails distributing a hybrid data-plus-code package which is more conducive to reuse, because the data set would contain speciliazed computer code optimized for accessing, viewing, and accessing the data provided.

The data-sharing and digital-text dimensions of software-enhanced publishing can also be interconnected, because publication texts will often refer to accompanying data sets (and vice-versa). Specific paragraphs, graphics figures, or technical terms within a publication can point to data fields or structures (e.g. a relational table, a data member of a class, or a procedure) belonging to dataset-specific computer code. Conversely, data-set applications (custom software for visualizing/utilizing particular data sets) may be enhanced with capabilities to load specific article text (e.g. by sending signals to PDF viewers) in response to users actions, such as right-clicking on a table column (assume a context menu item involves researching the terminology behind the column name, or its statistical properties/distribution, scientific/theoretical role, etc., where such explanations are provided at specific points in the publication). Both PDF viewers and custom data-set applications may be enhanced with granular cross-references between data sets and publication texts, which can be used to add special-purpose interactive features on both sides.
